# i18n (ES / EN)

## Decisión

- Idiomas soportados v0.1: **`es`** (default si `LANG` o flag lo indica), **`en`** (fallback).
- Resolución de locale: prioridad `NZ_MCP_LANG` env > `LANG` env > `en`.
- Mecanismo: catálogo en código (no `gettext` por simplicidad).

## Ámbito

| Texto | Idioma | Notas |
|---|---|---|
| Código fuente, nombres de funciones, comentarios | **inglés** | Siempre |
| Tool descriptions (lo que ve la IA) | **inglés** | Siempre |
| Mensajes de error al usuario | **i18n ES/EN** | Resolución por locale |
| Hints en respuestas | **i18n ES/EN** | |
| README.md | **español** | Principal |
| README.en.md | **inglés** | Paralelo |
| Docs internas (`AGENTS.md`, `docs/`) | **español** | Para revisión por owner ES |
| CHANGELOG.md | **bilingüe** | ES + EN por entrada |
| Commits, PRs, issues, reviews | **español** | |

## Estructura del catálogo

`src/nz_mcp/i18n.py`:

```python
from typing import Literal, TypedDict, Final

Locale = Literal["es", "en"]

class Message(TypedDict):
    es: str
    en: str

MESSAGES: Final[dict[str, Message]] = {
    "GUARD_REJECTED.UPDATE_REQUIRES_WHERE": {
        "es": "Las sentencias UPDATE requieren cláusula WHERE.",
        "en": "UPDATE statements require a WHERE clause.",
    },
    "GUARD_REJECTED.STACKED_NOT_ALLOWED": {
        "es": "No se permiten múltiples sentencias en una sola llamada.",
        "en": "Multiple statements in a single call are not allowed.",
    },
    # ...
    "HINT.RESULT_TRUNCATED_BY_ROWS": {
        "es": "Resultado truncado en {n} filas. Añade WHERE o LIMIT para refinar.",
        "en": "Result truncated at {n} rows. Add WHERE or LIMIT to refine.",
    },
}

def t(key: str, locale: Locale = "en", **fmt: object) -> str:
    msg = MESSAGES[key][locale]
    return msg.format(**fmt) if fmt else msg
```

## Reglas

- **Una clave, dos textos.** Si añades una clave en ES, debes añadirla en EN. Test que verifica paridad:

```python
def test_all_messages_have_both_locales():
    for key, msg in MESSAGES.items():
        assert "es" in msg, f"Missing ES for {key}"
        assert "en" in msg, f"Missing EN for {key}"
        assert msg["es"], f"Empty ES for {key}"
        assert msg["en"], f"Empty EN for {key}"
```

- **Claves namespaced**: `<DOMAIN>.<CODE>` (`GUARD_REJECTED.UPDATE_REQUIRES_WHERE`, `HINT.RESULT_TRUNCATED`).
- **Placeholders nombrados** (`{n}`, `{table}`), nunca posicionales.
- **No interpolar valores** dentro de los mensajes (sino con `.format()` controlado).
- **No traducir códigos** ni nombres de tools ni IDs.

## En el contrato de errores

Toda respuesta de error incluye:

```json
{
  "error": {
    "code": "GUARD_REJECTED",
    "message_es": "...",
    "message_en": "...",
    "hint_es": "..." ,
    "hint_en": "..."
  }
}
```

Ambos idiomas siempre presentes (no depender del locale del cliente).

## Logs

- Logs estructurados van en **inglés** (auditoría ops).
- El `code` de la excepción se usa para correlacionar; los `message_*` son para el cliente final.

## Anti-patrones

- ❌ Hardcodear strings en español dentro del código.
- ❌ Usar `gettext` para v0.1 (overkill).
- ❌ Mensajes con HTML o Markdown (los renderiza el cliente).
- ❌ Strings con concatenación: `"Tabla " + name + " no existe"`.
- ❌ Olvidar añadir EN cuando añades ES (o viceversa).
- ❌ Cambiar el texto de un código sin advertir en CHANGELOG (los `code` son estables, los `message_*` no — pero cambios cosméticos van en notas).

## Checklist

- [ ] Toda nueva clave en ES y EN.
- [ ] Test de paridad pasa.
- [ ] Errores devuelven ambos `message_es` y `message_en`.
- [ ] Hints en ambos idiomas.
- [ ] CHANGELOG menciona si añadí/renombré códigos.
